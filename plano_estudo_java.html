1. Tópicos Java — Guia Explicativo

1. JVM (Java Virtual Machine)

A JVM é a máquina virtual que executa o bytecode Java. Quando você compila um arquivo .java, o compilador (javac) gera arquivos .class contendo bytecode. A JVM interpreta e/ou compila esse bytecode em tempo de execução (JIT — Just-In-Time Compilation) para código nativo da máquina hospedeira.

Componentes principais: ClassLoader (carrega classes), Runtime Data Areas (Heap, Stack, Metaspace), Execution Engine e o Garbage Collector (GC), que gerencia a memória automaticamente (G1, ZGC, etc.).

2. Collections Framework

A biblioteca padrão do Java para estruturas de dados e algoritmos. Define interfaces cruciais como List (ArrayList, LinkedList), Set (HashSet, TreeSet), Map (HashMap, ConcurrentHashMap) e Queue.

Inclui contratos fundamentais como o relacionamento entre equals() e hashCode() e a poderosa Streams API (Java 8+) para operações funcionais.

3. POO (Programação Orientada a Objetos)

O paradigma central do Java sustentado por 4 pilares: Encapsulamento (proteger o estado com modificadores de acesso), Herança (reutilização e especialização), Polimorfismo (sobrecarga estática ou sobrescrita dinâmica) e Abstração (classes abstratas e interfaces).

Evoluções modernas incluem os Records (Java 16+) para dados imutáveis e Sealed Classes (Java 17+) para controle de hierarquias.

4. Spring Core

O alicerce do ecossistema Spring. Ele provê o container de Inversão de Controle (IoC) por meio do ApplicationContext e a Injeção de Dependência (DI). Ele gerencia o ciclo de vida e escopo dos beans (como Singleton e Prototype) e suporta AOP (Programação Orientada a Aspectos) para isolar transações e logs.

5. Spring Boot

Camada de opinião focada em produtividade que elimina o boilerplate de configuração. Traz o conceito de Autoconfiguração, dependências pré-configuradas (Starters), servidor embutido (Tomcat/Jetty), monitoramento em produção com o Spring Boot Actuator e gerenciamento por Profiles.

6. Pirâmide de Testes

Conceito de eficiência para qualidade de software: base larga composta de Testes Unitários rápidos (JUnit 5, Mockito), meio composto por Testes de Integração (Spring Boot Test, Testcontainers, WireMock) e o topo estreito com testes End-to-End (E2E) completos.

7. Spring Cloud

Conjunto de ferramentas para arquitetura de microsserviços e sistemas distribuídos. Inclui soluções para Configuração Centralizada (Config Server), Descoberta de Serviços (Eureka), API Gateway, Circuit Breaker (Resilience4j) e Rastreamento Distribuído (Micrometer + Zipkin).

8. Concorrência

Suporte nativo do Java para multiprocessamento através de threads tradicionais, utilitários de alto nível no pacote java.util.concurrent (como ExecutorService e CompletableFuture) e as modernas Virtual Threads (Java 21+) projetadas para alta escalabilidade de I/O.

9. Performance

Engloba técnicas de otimização voltadas a compiladores JIT, tuning de Garbage Collector, profiling e diagnósticos em tempo real utilizando Java Flight Recorder (JFR) e async-profiler, microbenchmarks com JMH e compilação nativa AOT com GraalVM.

10. Clean Code

Práticas de legibilidade e manutenibilidade popularizadas por Robert C. Martin. Enfatiza nomes significativos, funções de responsabilidade única, regras como DRY, KISS, YAGNI e o aprofundamento nos 5 princípios do SOLID.

11. Arquitetura de Software

Estrutura fundamental que governa a coesão, acoplamento e distribuição de componentes em um ecossistema. Compreende estilos que variam de Monolitos tradicionais a Microsserviços e Arquiteturas orientadas a eventos (Event-Driven), geralmente documentadas através de ADRs.

12. Arquitetura Hexagonal (Ports & Adapters)

Abordagem que coloca as regras de negócio (Domínio) no centro, isoladas do mundo externo. A comunicação ocorre por Interfaces chamadas de Ports (Inbound/Outbound) que são implementadas ou consumidas por Adapters (ex: REST Controllers, Repositórios JPA), garantindo que a tecnologia de infraestrutura mude sem afetar o negócio.

13. MVC (Model-View-Controller)

Padrão que divide as responsabilidades de entrada e representação de dados. No Spring MVC, o DispatcherServlet atua como centralizador de requisições, mapeando URIs para Controllers que processam regras e serializam dados em JSON ou Views de interface.

14. Clean Architecture

Estrutura em camadas concêntricas onde a regra de dependência dita que o código de fora conhece o de dentro, mas o inverso nunca ocorre. Divide o sistema estritamente em Entities, Use Cases (Casos de uso da aplicação), Interface Adapters e Frameworks externos.

15. Monolito Modular

Padrão de design que mantém o deploy unificado e simplificado de um monolito, mas estrutura o código internamente em módulos rígidos e isolados (Bounded Contexts). Ferramentas modernas como o Spring Modulith auxiliam a verificar e testar o acoplamento dessas fronteiras.

16. Integração com LLMs (LangChain4j)

Inclusão de inteligência artificial generativa em ecossistemas Java corporativos. O framework LangChain4j unifica conexões com modelos de IA (OpenAI, Claude, Ollama), gerencia padrões RAG (Busca semântica e Vector Stores) e provê recursos de Function Calling.

17. REST API

Estilo arquitetural baseado nos princípios do protocolo HTTP (Verbos, Recursos e Status Codes), sem retenção de estado (Stateless). Emprega práticas de paginação de dados, documentação via OpenAPI/Swagger e segurança via tokens JWT.

18. Mensageria

Comunicação assíncrona orientada a eventos ou tópicos para desacoplamento de microsserviços. Destaca o uso do Apache Kafka para logs de streaming distribuídos e de alta vazão e o RabbitMQ para roteamento de filas complexas via AMQP.
